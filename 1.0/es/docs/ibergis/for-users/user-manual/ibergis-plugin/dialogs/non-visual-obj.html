8.1.4. Diálogo Objetos no visuales

  • Curvas

  • Patrones

  • Series Temporales

  • Controles

  • Rásters

Herramienta para gestionar los objetos no visuales del proyecto.

Podemos gestionar curvas, patrones, series temporales, controles y rásters. En todos los casos, los siguientes botones están disponibles:

  • Duplicar: permite duplicar un objeto seleccionado.

  • Crear: permite crear un nuevo objeto.

  • Eliminar: permite eliminar un objeto seleccionado.

Esta ventana se utiliza para crear y configurar objetos no visuales. Estos elementos se almacenan en el modelo y pueden activarse selectivamente dependiendo de los requisitos de cada simulación.

8.1.4.1. Curvas

Permite definir curvas de tipo control, desviación, bomba, clasificación, forma, almacenamiento o mareal.

Para crear una curva tendremos que especificar pares de valores hasta que la curva esté definida. Dependiendo del tipo de curva, pueden ser necesarios varios pares de valores.

8.1.4.2. Patrones

Permite definir patrones temporales que podemos asignar a ciertos objetos de la red.

Podemos crear patrones diarios, horarios, mensuales o de fin de semana.

8.1.4.3. Series Temporales

Permite definir diferentes tipos de series temporales:

  • Elevación BC: condición de contorno para establecer una elevación específica de la superficie del agua.

  • Flujo BC: condición de contorno para establecer un flujo específico.

  • Evaporación: se puede cargar una serie temporal de tasas de evaporación para contabilizar la pérdida de agua debido a este factor durante simulaciones hidrológicas prolongadas.

  • Hidrograma de entrada: condición de contorno utilizada para definir una entrada específica a lo largo del tiempo usando un hidrograma (descarga vs tiempo). Esto permite modelar flujo variable que entra al dominio.

  • Orificio:

  • Precipitación: permite al usuario definir un hietograma para usarse en el modelo. Cada hietograma está asociado con coordenadas espaciales específicas y su ubicación puede ajustarse manualmente usando el editor de vértices, activando el modo de edición en QGIS y arrastrando el punto a la ubicación deseada. - Si solo se proporciona un hietograma, todo el dominio recibirá precipitación homogénea basada en ese hietograma. - Si se cargan múltiples hietogramas, la distribución de precipitación en el dominio se interpolará usando polígonos de Thiessen.

  • Temperatura: se puede cargar una serie temporal de temperatura.

Excepto para la opción BC ELEVATION, en todos los demás casos los datos pueden definirse manualmente (usando tiempos relativos o absolutos) o importándolos desde un archivo externo.

Para asignar cualquiera de las series temporales previamente definidas al modelo, debe especificar la ubicación o geometría donde se aplicará la serie. Esto puede hacerse a través de la tabla de atributos activando el modo de edición y seleccionando la geometría relevante. La tabla de atributos muestra una fila para cada geometría o ubicación en el dominio. Para asignar la serie temporal apropiada a cada ubicación, active el modo de edición y use el menú desplegable en la columna «timeseries» para elegir entre las series temporales predefinidas.

8.1.4.4. Controles

Órdenes o instrucciones que determinan cómo funciona la red a lo largo del tiempo.

8.1.4.5. Rásters

Integran datos espaciales para representar la variabilidad de parámetros como la precipitación.

Estos rásters pueden representar ya sea volumen de precipitación (mm) o intensidad de precipitación (mm/h). El usuario debe especificar el tipo de datos contenidos en los rásters, así como la marca de tiempo correspondiente para cada archivo para asegurar la asignación temporal correcta durante la simulación.

Nota

Los rásters pueden importarse con el botón de importación en el gestor.

Importante

Los objetos no visuales deben asignarse a sus ubicaciones correspondientes (geometrías) en el modelo a través de sus respectivas tablas de atributos.
