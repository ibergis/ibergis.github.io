Herramienta para gestionar todas las opciones involucradas en la simulación. Podemos gestionar las opciones de SWMM e Iber.

8.2.1.1. Pestaña OPCIONES DE IBERGIS

  • Nombre del proyecto: nombre del proyecto actual.

  • Descripción: una breve descripción del proyecto.

  • Usuario: nombre del usuario o autor del proyecto.

  • Fecha de creación: fecha de creación del proyecto.

  • Versión: versión de IberGIS.

8.2.1.2. Pestaña OPCIONES DE SWMM

Las siguientes opciones están disponibles:

  • Omitir estado estacionario: permite omitir la fase estacionaria al comienzo de la simulación hidráulica. El valor predeterminado es NO.

  • Permitir encharcamiento: determina si se permite que el agua en exceso se acumule sobre los nodos y se reintroduzca en el sistema cuando las condiciones lo permitan. El valor predeterminado es SÍ. Para que el encharcamiento realmente ocurra en un nodo particular, debe usarse un valor distinto de cero para su atributo de área encharcada (campo apond en la capa Junction).

  • Pendiente mínima: valor mínimo permitido para la pendiente de un conducto (%). Si es cero (el valor predeterminado) entonces no se impone ningún mínimo (aunque SWMM usa un límite inferior de caída de elevación de 0.00035 m al calcular la pendiente de un conducto).

  • Unidades de flujo: permite elegir las unidades de flujo. El valor predeterminado es CMS.

  • Enrutamiento de flujo: permite elegir qué método se usa para enrutar flujos a través del sistema de drenaje. El valor predeterminado es DYNWAVE.

  • Desplazamientos de enlace: determina la convención usada para especificar la posición de un desplazamiento de enlace sobre la cota inferior de su nodo de conexión. DEPTH indica que los desplazamientos se expresan como la distancia entre la cota inferior del nodo y el enlace, mientras que ELEVATION indica que se usa la elevación absoluta del desplazamiento. El valor predeterminado es ELEVATION.

  • Ignorar agua subterránea: permite elegir si los cálculos de agua subterránea deben ignorarse cuando un archivo de proyecto contiene objetos acuíferos. El valor predeterminado es NO.

  • Ignorar precipitación: permite elegir si todos los datos de precipitación y cálculos de escorrentía deben ignorarse. Si se establece en SÍ, SWMM solo realiza enrutamiento de flujo y contaminantes basado en entradas directas y de clima seco proporcionadas por el usuario. El valor predeterminado es NO.

  • Ignorar enrutamiento: permite elegir si solo debe calcularse la escorrentía incluso si el proyecto contiene enlaces y nodos del sistema de drenaje. El valor predeterminado es NO.

  • Ignorar deshielo: permite elegir si los cálculos de deshielo deben ignorarse cuando un archivo de proyecto contiene objetos de paquete de nieve. El valor predeterminado es NO.

  • Ignorar calidad: permite elegir si el lavado de contaminantes, enrutamiento y tratamiento deben ignorarse en un proyecto que tiene contaminantes definidos. El valor predeterminado es NO.

  • Ecuación de conducción forzada: establece si se usará la ecuación de Hazen-Williams (H-W) o la de Darcy-Weisbach (D-W) para calcular las pérdidas por fricción para flujo presurizado en conductos que tienen asignada una forma de sección transversal circular de conducción forzada. El valor predeterminado es H-W.

  • Amortiguamiento inercial: indica cómo se manejarán los términos inerciales en la ecuación de momento de Saint Venant bajo enrutamiento de onda dinámica. Elegir NONE (el valor predeterminado) mantiene estos términos en su valor completo bajo todas las condiciones. Seleccionar PARTIAL reducirá los términos a medida que el flujo se acerca a ser crítico (y los ignora cuando el flujo es supercrítico). Elegir FULL eliminará los términos por completo.

  • Tolerancia de flujo lateral: diferencia porcentual máxima entre la entrada lateral actual y la anterior en todos los nodos del sistema de conducción para que la opción Omitir estado estacionario surta efecto. El valor predeterminado es 5 por ciento.

  • Paso de alargamiento: paso de tiempo, en segundos, usado para alargar conductos bajo enrutamiento de onda dinámica, de modo que cumplan el criterio de estabilidad de Courant bajo condiciones de flujo completo (es decir, el tiempo de viaje de una onda no será menor que el paso de tiempo de alargamiento de conducto especificado). A medida que este valor disminuye, menos conductos requerirán alargamiento. Un valor de 0 significa que no se alargarán conductos. El valor predeterminado es 1.

  • Paso húmedo: longitud de paso de tiempo usada para calcular la escorrentía de subcuencas durante períodos de lluvia o cuando el agua encharcada aún permanece en la superficie. El valor predeterminado es 00:05:00.

  • Inicio de barrido: día del año (mes/día) cuando comienzan las operaciones de barrido de calles. El valor predeterminado es 01/01.

  • Fin de barrido: día del año (mes/día) cuando terminan las operaciones de barrido de calles. El valor predeterminado es 12/31.

  • Paso variable: factor de seguridad aplicado a un paso de tiempo variable calculado para cada período de tiempo bajo enrutamiento de onda dinámica. El paso de tiempo variable se calcula para satisfacer el criterio de estabilidad de Courant para cada conducto y aún así no exceder el valor de Paso de enrutamiento. Si el factor de seguridad es 0, entonces no se usa paso de tiempo variable. El valor predeterminado es 0.75.

  • Flujo normal limitado: especifica qué condición se verifica para determinar si el flujo en un conducto es supercrítico y por lo tanto debe limitarse al flujo normal. Use SLOPE para verificar si la pendiente de la superficie del agua es mayor que la pendiente del conducto, FROUDE para verificar si el número de Froude es mayor que 1 o BOTH para verificar ambas condiciones. El valor predeterminado es BOTH.

  • Área superficial mínima: área superficial mínima usada en los nodos al calcular cambios en la profundidad del agua bajo enrutamiento de onda dinámica. Si se ingresa 0, entonces se usa el valor predeterminado de 1.167 m² (es decir, el área de un pozo de inspección de 1,2192 m de diámetro).

  • Días secos: número de días sin lluvia antes del inicio de la simulación. El valor predeterminado es 10.

  • Paso seco: longitud de paso de tiempo usada para cálculos de escorrentía (que consiste esencialmente en acumulación de contaminantes) durante períodos cuando no hay lluvia ni agua encharcada. El valor predeterminado es 1:00:00.

  • Tolerancia de carga: diferencia en la carga calculada en cada nodo entre intentos sucesivos por debajo de la cual se asume que la solución de flujo para el paso de tiempo actual ha convergido. La tolerancia predeterminada es 0.

  • Intentos máximos: número máximo de intentos permitidos durante un paso de tiempo para alcanzar convergencia al actualizar las cargas hidráulicas en los nodos del sistema de conducción. El valor predeterminado es 0.

  • Tolerancia de flujo del sistema: diferencia porcentual máxima entre la entrada total del sistema y la salida total del sistema que puede ocurrir para que la opción Omitir estado estacionario surta efecto. El valor predeterminado es 5 por ciento.

  • Hilos: número de hilos de computación paralela a usar para enrutamiento de flujo de onda dinámica en máquinas equipadas con procesadores multinúcleo. El valor predeterminado es 1.

  • Paso mínimo: el paso de tiempo más pequeño permitido cuando se usan pasos de tiempo variables para enrutamiento de flujo de onda dinámica. El valor predeterminado es 0.5.

  • Paso de enrutamiento: longitud de paso de tiempo en segundos usada para enrutar flujos y constituyentes de calidad del agua a través del sistema de conducción. Esto puede aumentarse si no se usa enrutamiento de onda dinámica. El valor predeterminado es 00:00:02.

  • Fecha de inicio: la fecha cuando comienza la simulación. Si no se proporciona, se usa la fecha 01/01/2017.

  • Fecha de fin: la fecha cuando debe terminar la simulación. El valor predeterminado es la fecha de inicio.

  • Fecha de inicio de informe: la fecha cuando debe comenzar el informe de resultados. El valor predeterminado es la fecha de inicio de la simulación.

  • Paso de informe: intervalo de tiempo para informar resultados calculados. El valor predeterminado es 00:05:00.

  • Hora de inicio: hora del día en la fecha de inicio cuando comienza la simulación. El valor predeterminado es 00:00:00.

  • Hora de fin: hora del día en la fecha de finalización cuando terminará la simulación. El valor predeterminado es 03:00:00.

  • Hora de inicio de informe: hora del día en la fecha de inicio del informe cuando debe comenzar el informe. El valor predeterminado es la hora de inicio de la simulación.

8.2.1.3. Pestaña RESULTADOS DE SWMM

Las siguientes opciones están disponibles:

  • Continuidad: especifica si las verificaciones de continuidad deben reportarse o no. El valor predeterminado es SÍ.

  • Estadísticas de flujo: especifica si las estadísticas resumidas de flujo deben reportarse o no. El valor predeterminado es SÍ.

  • Controles: especifica si todas las acciones de control tomadas durante una simulación deben listarse o no. El valor predeterminado es SÍ.

  • Entrada: especifica si debe proporcionarse o no un resumen de los datos de entrada en el informe de salida. El valor predeterminado es NO.

  • Subcuencas detalladas por paso de tiempo: lista de subcuencas cuyos resultados deben reportarse. El valor predeterminado está en blanco.

  • Nodos detallados por paso de tiempo I y II: lista de nodos cuyos resultados deben reportarse. El valor predeterminado está en blanco. Se puede escribir un máximo de 40 nodos en cada uno de ellos.

  • Enlaces detallados por paso de tiempo: lista de enlaces cuyos resultados deben reportarse. El valor predeterminado está en blanco.

8.2.1.4. Pestaña OPCIONES DE IBER

Las siguientes opciones están disponibles:

Esquema numérico

  • Esquema numérico: define el esquema numérico para las ecuaciones de Saint-Venant usadas en el cálculo del flujo superficial. Las opciones disponibles son 1er Orden, 2do Orden, DHD y DHD Basin. El valor predeterminado es DHD.

  • CFL: condición de Courant-Friedrichs-Lewy. El usuario debe establecer el valor. Si se establece un valor muy alto, el tiempo de cálculo se reducirá, pero pueden surgir problemas de convergencia en la solución de las ecuaciones. Un valor de 0.45 puede considerarse apropiado como punto de partida, y puede reducirse a 0.3 si se observan problemas de convergencia.

  • Incremento de tiempo máximo: establece el valor máximo del paso de tiempo usado por el programa para integrar las ecuaciones de flujo. Este es un valor máximo. En la práctica, el paso de tiempo usado durante la simulación será el mínimo entre este valor y el calculado basándose en la condición CFL. El valor predeterminado es 1.

  • Límite húmedo-seco: este es el umbral de profundidad por encima del cual un elemento se considera húmedo. Por debajo de este valor, el elemento se considera seco, y por lo tanto no se realizan cálculos en él, a menos que se moje. Mientras que en ingeniería de ríos, un umbral de 0.01 m es generalmente razonable, en algunos casos este valor podría reducirse hasta 0.001 m. El valor predeterminado es 0.0001.

  • Viscosidad molecular: coeficiente de viscosidad molecular aplicado al flujo para representar efectos de difusión numérica. El valor predeterminado es 0.000001.

Importante

Los esquemas DHD y DHD Basin no deben usarse para simulaciones hidráulicas

Control de Tiempo y Simulación

  • Intervalo de tiempo de resultados 2D: intervalo de tiempo, en segundos, que define cada cuánto se guarda un resultado 2D durante la simulación. Esto debe coincidir con el valor de Paso de informe. El valor predeterminado es 300.

  • Intervalo de tiempo de series temporales: intervalo de tiempo, en segundos, entre registros de datos en las series temporales. Si no está definido o es mayor que el Intervalo de tiempo de resultado 2D, se usa ese valor. El valor predeterminado es 300.

Procesos hidrológicos

  • Tiempo de inicio: tiempo de inicio de la simulación, en segundos. El valor predeterminado es 0.

  • Precipitación: selecciona el tipo de precipitación aplicada en el modelo. Las opciones disponibles son Hietograma, Ráster y Sin lluvia. El valor predeterminado es Sin lluvia.

  • Establecer ráster de precipitación: permite elegir entre los rásters de precipitación definidos.

  • Método de pérdidas: método usado para calcular pérdidas por infiltración. El valor predeterminado es SCS.

  • Multiplicador CN: factor multiplicador aplicado al Número de Curva para ajustar las pérdidas de escorrentía. El valor predeterminado es 1.

  • Ia: abstracción inicial. Relación de almacenamiento antes del inicio del flujo. El valor predeterminado es 0.2.

8.2.1.5. Pestaña RESULTADOS DE IBER

Las siguientes opciones están disponibles:

Hidrodinámica

  • Profundidad: resultados de profundidad del agua, en metros, en cada celda de la malla a lo largo del tiempo.

  • Velocidad: resultado de velocidad del flujo, en m/s, en cada celda de la malla a lo largo del tiempo.

  • Caudal específico: resultado de caudal específico, en m²/s

  • Elevación del agua: resultado del nivel total del agua, en metros, la suma de la elevación del terreno más la profundidad del agua.

Máximos

  • Profundidad máxima: profundidad máxima del agua, en metros, alcanzada en cada celda durante la simulación.

  • Velocidad máxima: velocidad máxima del flujo, en m/s, alcanzada en cada celda durante la simulación.

  • Caudal específico máximo: caudal específico máximo, en m²/s, durante la simulación.

  • Elevación máxima del agua: nivel máximo total del agua, en metros, durante la simulación.

Otros resultados

  • Energía: representa la energía total del flujo, calculada como la suma del nivel del agua y la energía cinética.

  • Número de Froude: número de Froude que indica el régimen de flujo hidráulico: subcrítico (Fr<1), crítico (Fr=1) o supercrítico (Fr>1).

  • Paso de tiempo local: paso de tiempo local calculado según la condición de estabilidad de Courant-Friedrichs-Lewy (CFL).

  • Paso de tiempo local máximo: valor máximo permitido del paso de tiempo local en cada celda del modelo, que limita el incremento de tiempo calculado según la condición de Courant.

Peligrosidad

  • Peligrosidad RD9/2008: criterio de peligrosidad hidráulica definido por el Ministerio Español de Medio Ambiente, que clasifica el riesgo basándose en la profundidad del agua y la velocidad del flujo.

  • Peligrosidad ACA 2003: criterio de peligrosidad de la Agencia Catalana del Agua (2003), que clasifica zonas de riesgo basándose en la profundidad del agua y la velocidad del flujo.

  • Peatones: evalúa los peligros del flujo para peatones, considerando la profundidad y velocidad del agua como factores determinantes de la estabilidad (criterio UPC).

  • Vehículos: evalúa la seguridad de los vehículos contra el arrastre por corrientes, basándose en parámetros hidráulicos como la profundidad del agua y la velocidad del flujo (criterio australiano (Shand et al., 2011)).

Resultados ráster

  • Resultados ráster: método de interpolación para resultados ráster. Las opciones disponibles son Interpolación lineal e Interpolación del vecino más cercano. El valor predeterminado es Interpolación lineal.

  • Tamaño de celda: tamaño de celda de los rásters de salida, en metros. El valor predeterminado es 100.

  • Máximos al final: añade un paso de tiempo más al final de los rásters de resultado con los valores máximos obtenidos.

  • Usar marco ráster: permite delimitar el área geográfica donde se generarán los resultados ráster. Los parámetros se definen en el siguiente grupo, Marco de resultados ráster.

Marco de resultados ráster

Permite especificar las coordenadas del límite del ráster.

8.2.1.6. Pestaña PLUGINS DE IBER

Las siguientes opciones están disponibles:

  • Solo entradas o red completa: permite elegir si simular solo la escorrentía superficial (solo entradas) o incluir la red de drenaje completa (red completa). El valor predeterminado es Red completa.

  • Habilitar o deshabilitar pérdida de salida: habilita o deshabilita la consideración de pérdidas de energía en las salidas del modelo.

Pestaña VERIFICAR PROYECTO

Establece valores máximos y mínimos que pueden incluirse o excluirse, para parámetros: manning, cellsize, sfactor, slope, street_vol, orifice_cd, roughness, mfactor, ufactor, outlet_vol y weir_cd.

Pestaña OPCIONES DE RÁSTER

Las siguientes opciones están disponibles:

Modo de simbología de resultados ráster

Establece el modo de simbología de resultados ráster. Las opciones disponibles son:

  • Mín/Máx por banda: ajusta automáticamente la escala de color para cada banda ráster individualmente usando los valores mínimo y máximo detectados dentro de esa banda.

  • Mín/Máx para todas las bandas: aplica una sola escala de color a todas las bandas ráster, usando los valores mínimo y máximo globales encontrados en todas las bandas.

  • Mín/Máx personalizado: usa valores mínimo y máximo personalizados definidos en Configuración ráster para cada capa para todas las bandas ráster.

Configuración ráster para cada capa

Permite configurar los resultados ráster para cada capa. Las siguientes opciones están disponibles:

  • Valores personalizados máximo y mínimo: establece los valores máximo y mínimo para los resultados ráster.

  • Rampa de color de simbología: establece la rampa de color de simbología.

  • Excluir valores para los parámetros: permite excluir valores para facilitar la configuración.

Las capas disponibles para configurar son:

  • Profundidad

  • Velocidad

  • Elevación del Agua

  • Peligrosidad ACA

  • Peligrosidad Grave RD9-2008

  • Paso de Tiempo Local

  • Caudal Específico X

  • Caudal Específico Y

  • Energía

  • Froude

  • Tasa de Infiltración

  • Profundidad de Lluvia

  • Velocidad X

  • Velocidad Y

  • Rendimiento del Agua

  • Profundidad Máxima

  • Velocidad Máxima

  • Elevación Máxima del Agua

  • Peligrosidad ACA Máxima

  • Peligrosidad Grave RD9-2008 Máxima

  • Paso de Tiempo Local Máximo

  • Caudal Específico Máximo

Importante

La configuración de máximo y mínimo solo funcionará con el modo de simbología personalizado.

Nota

La ejecución puede generar más rásters que los disponibles para configurar en este diálogo. También se importarán, pero con configuración predeterminada.
